# a0 Roadmap

This roadmap is a practical contributor guide, not a promise of dates.

> Naming: `a0` is the project / runtime described by this roadmap. `a0p` is the deployed instance of `a0` running publicly. See "Project name: `a0` vs `a0p`" in `README.md`.

## Phase 1 — Make the project legible

- [ ] Confirm canonical repository structure.
- [ ] Document local setup.
- [ ] Document runtime entry points.
- [ ] Add architecture diagram.
- [ ] Create issue templates.
- [ ] Label contributor-friendly issues.

## Phase 2 — Stabilize implementation boundaries

- [ ] Define a0 agent runtime responsibilities.
- [ ] Define AIMMH provider-routing responsibilities.
- [ ] Draft a0 ↔ AIMMH API schema.
- [ ] Document tool execution permissions.
- [ ] Document memory/context behavior.

## Phase 3 — Improve developer experience

- [ ] Add one-command setup if possible.
- [ ] Add smoke tests.
- [ ] Add CI checks.
- [ ] Add example configuration without secrets.
- [ ] Add troubleshooting guide.

## Phase 4 — Build evaluation harness

- [ ] Add provider routing tests.
- [ ] Add tool-call schema tests.
- [ ] Add instruction-following regression tests.
- [ ] Add safety/audit behavior tests.
- [ ] Add website build checks.

## Phase 5 — Recruit and coordinate contributors

- [ ] Publish help-wanted issue.
- [ ] Share contributor landing page in relevant developer communities.
- [ ] Invite architecture review from agent-framework builders.
- [ ] Keep issues small enough for new contributors.

## Current priority

The highest priority is making a fresh contributor able to understand what exists, run it locally, and choose a small useful task.
