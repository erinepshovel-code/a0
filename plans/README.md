# plans

This folder is the coordination surface for repository-level planning across The Interdependency projects.

Every repo plan should answer the same three questions:

1. **What done looks like** — the minimum condition where the repo can be treated as coherent, usable, and ready for its current role.
2. **Current state** — what appears true now, including uncertainty.
3. **Likeliest next actions** — the next small moves most likely to improve the repo.

## Folder update contract

When any repository status changes, update the matching file in this folder.

When a new repository becomes part of the system, add a new file here using this naming pattern:

```text
plans/<repo_name>_plan.md
```

Use lowercase names. Replace hyphens with underscores when needed.
