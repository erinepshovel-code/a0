# a0p Fleet & Run Lifecycle

The Fleet is a0p's observability spine. Every multi-step, multi-provider, or
multi-agent operation is a **run** with a stable id, a parent, a depth, a
status, a cost ledger, and a stream of structured events. The Fleet UI
(`client/src/pages/fleet.tsx`) is the user-facing window onto that data.

This skill defines the contracts. Break them and the Fleet shows phantom
runs, leaks SSE connections, or hides errors that the user paid for.

---

## 1. Data model — `agent_runs` and `agent_logs`

Schemas live in `python/main.py` (around the boot DDL block). Do not
duplicate the DDL elsewhere; extend it there if a new column is truly
required, then update `_row_to_run` in `python/routes/runs.py` so the new
field reaches the UI.

`agent_runs`

| column              | rule |
|---------------------|------|
| `id`                | UUID string. Generated by the spawner, never reused. |
| `parent_run_id`     | NULL only for top-level runs (user message → primary). |
| `root_run_id`       | Equal to `id` for roots; otherwise the root of the tree. |
| `depth`             | 0 for roots, parent.depth + 1 for children. Enforced by `spawn_caps`. |
| `status`            | `running` → terminal one of `completed` / `failed` / `killed`. No other values. |
| `orchestration_mode`| `single` / `fan_out` / `council` / `daisy_chain`. Lowercase, snake_case. |
| `cut_mode`          | `off` / `soft` / `hard`. |
| `providers`         | JSONB array of provider ids (`["gemini", "grok", ...]`). Empty for `single`. |
| `spawned_by_tool`   | Tool name that spawned the run, or NULL for root. Used as fallback display label. |
| `task_summary`      | Short human-readable string. Becomes the row title in the Fleet. |
| `total_tokens` / `total_cost_usd` | Aggregated *only* by run-finalization code, not by per-event emitters. |

`agent_logs`

| column          | rule |
|-----------------|------|
| `id`            | UUID per event. |
| `run_id`        | Must reference a real `agent_runs.id`. Use `"_orphan"` only if context is genuinely missing. |
| `depth` / `parent_run_id` | Snapshotted from ContextVars at emit time, not re-read at flush time. |
| `level`         | One of `INFO` / `WARN` / `ERROR` / `DEBUG`. Capitalized in the table; the UI lowercases. |
| `event`         | One of the enum below. |
| `payload`       | JSONB. Keep small (<4 KB). Distill long tool outputs first. |

## 2. Event vocabulary

`python/services/run_logger.py::_VALID_EVENTS` is the single source of truth:

```
spawn_start, spawn_complete, tool_call, tool_result,
merge, cap_hit, provider_response, error, custom
```

Anything outside this set is silently rewritten to `custom`. To add a new
event name, edit `_VALID_EVENTS` *and* mention it in this skill so future
agents know the new vocabulary exists. Do not invent event names ad-hoc —
the Fleet color/filter logic depends on the enum.

Payload conventions per event:

- `spawn_start`: `{ "tool": ..., "task": ..., "providers": [...] }`
- `spawn_complete`: `{ "duration_ms": N, "ok": true|false }`
- `tool_call`: `{ "tool": ..., "args_summary": "...", "args_bytes": N }`
- `tool_result`: `{ "tool": ..., "ok": true|false, "result_bytes": N, "distilled": true|false }`
- `merge`: `{ "into_run": "...", "tokens": N }`
- `cap_hit`: `{ "kind": "depth"|"breadth"|"cost"|"tokens", "limit": N, "actual": N, "tier": "free"|"supporter"|"ws" }`
- `provider_response`: `{ "provider": ..., "model": ..., "tokens": N, "cost_usd": N, "latency_ms": N }`
- `error`: `{ "where": "...", "type": "...", "message": "..." }` — never the full traceback; log it separately if needed.

## 3. ContextVars — `python/services/run_context.py`

Run lineage is propagated via ContextVars, never via function arguments
threaded through every call site. The canonical accessors:

```python
get_current_run_id() -> str | None
get_current_parent_run_id() -> str | None
get_current_root_run_id() -> str | None
get_current_depth() -> int
```

Rules:

1. Set them at exactly **one** place per run: the spawner. For sub-agents
   that's `services/tools/sub_agent_spawn.py`; for primary runs it's the
   chat send path. Use a try/finally with a `Token` so reset always runs.
2. **Never** call `set` without a matching `reset`. The Fleet will
   misattribute every subsequent log if the var leaks across requests.
3. Read them with `get_*()`; do not import the underlying ContextVar
   directly. The accessor functions are the public API.
4. Async work that escapes the spawner's scope (background tasks,
   `asyncio.create_task`) must capture the values *before* scheduling and
   re-set them inside the task using `contextvars.copy_context().run(...)`.

## 4. Logging — `run_logger.emit`

```python
from python.services.run_logger import get_run_logger
get_run_logger().emit("tool_call", {"tool": "bash_run", "args_bytes": len(payload)})
```

- `emit` is fire-and-forget. It enqueues an in-memory row and triggers a
  flush if the queue exceeds 100 entries. A 1 s background flusher drains
  the rest. Never block on the result.
- `emit` simultaneously calls `routes.runs.publish_log` so SSE subscribers
  see the event immediately. If `publish_log` raises, `emit` swallows it
  (we will not let observability fail a run).
- DB write failures inside `flush` are **re-raised**, not swallowed. The
  buffer is preserved and the next flush retries. This is intentional and
  must not be changed — a silent fallback would let runs go dark.
- Always end a run with a terminal-state DB update (`agent_runs.status`)
  followed by `await flush()`. Without the explicit flush, the SSE
  heartbeat may report the terminal status before the last `tool_result`
  reaches the table, and the UI will show an inconsistent ladder.

## 5. SSE contract — `/api/v1/runs/{id}/logs/stream`

Server emits **named** events, not default `message`:

```
event: log
data: { ...full log row... }

event: heartbeat
data: { "status": "running" | "completed" | "failed" | "killed" | "missing" }
```

Client rules (see `client/src/pages/fleet.tsx`):

1. **Use `addEventListener("log", ...)` and `addEventListener("heartbeat", ...)`.**
   Native `EventSource.onmessage` only fires on the unnamed default event
   and will silently drop everything the server sends.
2. Treat any non-`running`/`queued`/`pending` heartbeat as terminal,
   close the EventSource, and stop reconnecting. Without this the browser
   will reconnect every few seconds against a finished run forever.
3. On a clean component unmount call `es.close()`. The server unsubscribes
   on disconnect, but lingering client connections still consume a queue
   slot in `_SSE_QUEUES`.
4. Cap the in-memory log list (~500 entries). The server backfills the
   most recent 25 events on connect, so users still see context after a
   refresh; the UI does not need to keep an unbounded history.

To add a new SSE event type, add it on the server in `runs.py` (`yield
f"event: ..."`), document it here, and register the listener on the client.
Renaming an event is a breaking change for every connected browser.

## 6. Spawn caps — `python/services/spawn_caps.py`

Every sub-agent spawn must call the cap check before creating the
`agent_runs` row:

- `max_depth` — recursion limit per tier
- `max_breadth` — concurrent children per parent
- `max_cost_usd` / `max_tokens` per root tree

When a cap trips, emit a `cap_hit` event (so the user sees *why* their
fleet stopped expanding) and raise the dedicated `SpawnCapExceeded`
exception. Do **not** silently downgrade to a smaller orchestration —
that hides cost-control behavior the user explicitly configured.

## 7. Adding a new orchestration mode

1. Add the mode id to `MODES` in `client/src/components/chat-input.tsx`
   (`{ id, label, multi }`). `multi: true` exposes the provider chip
   picker.
2. Implement the dispatcher branch in
   `python/services/inference_modes.py::run_inference_with_mode`.
3. The dispatcher must populate `usage["responses"]` as a list of
   `{provider, model, content, error?, usage?}` dicts. The Fleet UI
   renders this list; `daisy_chain` and `daisy-chain` get the ladder
   layout, everything else gets the side-by-side card grid.
4. Always set `usage["orchestration_mode"]` to the mode id so the UI
   label stays accurate.
5. Emit one `provider_response` event per provider call so the Fleet log
   tail tells the same story the message bubble does.

## 8. Adding a new tool that spawns sub-runs

Mandatory checklist:

- [ ] Generate a UUID `id` and insert into `agent_runs` *before* you do
      any work. Status starts as `running`.
- [ ] Set `parent_run_id`, `root_run_id`, `depth` from ContextVars.
- [ ] Wrap the body in try / except / finally that updates `status` to
      `completed` / `failed` and sets `ended_at`.
- [ ] Emit `spawn_start` immediately and `spawn_complete` (or `error`)
      at the end.
- [ ] Aggregate `total_tokens` / `total_cost_usd` once at finalization,
      not per event.
- [ ] `await flush()` before returning to the caller.

Reference implementation: `python/services/tools/sub_agent_spawn.py` and
`sub_agent_merge.py`. Copy that shape; do not invent a new lifecycle.

## 9. Cost & token attribution

- Per-call cost is computed by the provider adapter (`python/services/providers/*`)
  and returned in the response. It is the spawner's job to add it to
  `agent_runs.total_cost_usd` and to emit a `provider_response` event with
  the `cost_usd` field populated.
- The Fleet header shows `cost_today_usd` summed across **all** runs in
  the `/tree` response (currently capped at 500 rows). If you add a
  retention policy that prunes rows, surface a stat that reflects that or
  the number will silently drift.

## 10. Common mistakes the Fleet exposes

- **Phantom roots** — A run row appears with no parent because the tool
  forgot to inherit `parent_run_id` from ContextVars. Fix: use the
  `get_current_*` accessors, not `None`.
- **Frozen "running" state** — Status never updated because the finally
  block raised. Always wrap the status update in its own try.
- **Empty log tail** — The new tool emits to `print` instead of
  `run_logger.emit`. Migrate the log lines.
- **SSE flickers `disconnected`** — Component re-renders are recreating
  the EventSource. Memoize `runId` or move the effect into a stable child.
- **Unattributed cost** — Provider adapter returned cost but the spawner
  did not add it to `total_cost_usd`. The Fleet stat undercounts.
